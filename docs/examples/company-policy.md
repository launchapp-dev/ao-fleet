# Company Policy

`ao-fleet` is the company control plane.

The company layer decides:

- which AO teams exist
- which repos they own
- when teams should run
- which policies apply across the fleet
- which knowledge records should be promoted to company memory

Default operating rules:

- marketing teams stay on business-hours schedules
- product teams can use nightly schedules if daytime work is disruptive
- platform teams keep burst-on-backlog or manual control when they own shared runtime work

This file is the source note used by the example company snapshot.
